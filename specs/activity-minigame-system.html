Purpose

Extend the shared simulation from a simple presence indicator (which room are you in?) into a richer ambient experience: players explore within their current room, optionally start mini-games, and those activities animate on the partner's screen in real time. Room transitions remain exclusively physical (NFC board), preserving the tactile gift metaphor. A new "work" location adds a scripted commute sequence.

Problem

The current simulation only communicates which room a partner is in. There is no depth within a room — no activities, no animations beyond walking to a room center, and no way to express "what" the partner is doing. The "going to work" scenario is completely absent. Adding mini-games and new locations piecemeal without a registry system would require touching many files for each new game.

Solution

Introduce a two-level state: location (set exclusively by NFC) and activity (set digitally within the current location). Each activity has a self-animation (what you see on your screen) and a partner-animation (what your partner sees on their screen). All activities and locations are registered in a single config file — adding a new mini-game or POI requires one config entry and one Phaser Scene, nothing else. A dedicated "work" pipeline handles the commute: NFC chip → driving scene on your screen → "Arrived" button → work scene on your screen + "at work" on partner's screen.

5.2. AnimationPlayer System

Create src/systems/AnimationPlayer.ts:

export class AnimationPlayer {
  private _timer: Phaser.Time.TimerEvent | null = null;
  private _loop = false;

  constructor(private scene: Phaser.Scene, private sim: Sim) {}

  // Play a sequence of steps in order. Cancels any running sequence.
  play(seq: AnimSequence, stepIndex = 0): void {
    this.stop();
    const step = seq[stepIndex];
    if (!step) return;
    if (step.moveTo) this.sim.walkTo(step.moveTo.x, step.moveTo.y);
    this.sim.playAnim(step.anim);
    if (step.loop) { this._loop = true; return; }
    this._timer = this.scene.time.delayedCall(step.duration, () => {
      this.play(seq, stepIndex + 1);
    });
  }

  stop(): void {
    this._timer?.destroy(); this._timer = null; this._loop = false;
  }
}

Notes

Design Principle — Physical Gating Room transitions are exclusively triggered by NFC. The digital world never moves a sim to a new room via click or keyboard. This preserves the physical metaphor: you must physically move the figurine to change rooms. Only in-room activities are digital. This is a deliberate constraint; do not add a "click to move room" fallback even in dev/testing — use PocketBaseService.publishPosition() via browser console for manual testing.
Mini-game Scene Lifecycle Mini-game scenes (MagazineMayhemScene etc.) must never directly call PocketBase. All state publish/subscribe flows through ActivitySystem. This keeps scenes testable in isolation and ensures the system stays coherent if a scene crashes or the user force-closes it.

Adding a New Mini-game — Checklist

This is the entire dev workflow for a new mini-game (no other files need touching):

  1. Create src/scenes/minigames/YourGameScene.ts with a unique key
  2. Register the scene key in main.ts scene array
  3. Add an entry to ACTIVITIES in src/config/activities.ts — set rooms, selfSceneKey, and partnerAnim
  4. Add any new sprite animation definitions to BootScene.ts (anims.create())
  5. Done. The POI overlay and partner animation will pick it up automatically.

Adding a New Location

For locations beyond the 7 planned (6 house rooms + work):

  1. Add hardware NFC reader on the next mux channel
  2. Extend ROOM_NAMES in nfc_grid.cpp
  3. Add the new location to LocationName union in rooms.ts
  4. Add an entry to ROOM_ENTER_ANIMS in room-enter-anims.ts
  5. If it has a special scene (like work): add handling in ActivitySystem.onRoomChange()
  6. If it's a regular house room: add coordinates to ROOMS and update house.png

Partner-side "At Work" Display

When a partner is at room: 'work', the house view doesn't have a natural place to show them. Implementation decision: render a DOM overlay card in the bottom-right corner of the game canvas showing a status badge ("Partner is commuting 🚗" or "Partner is at work 💼"). The partner sim Sim GameObject is hidden (setVisible(false)) while this card is shown, and restored on return home.
